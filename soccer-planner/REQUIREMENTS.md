# 📝 Project Requirements & Roadmap

본 프로젝트는 단순한 포지션 배정 도구를 넘어, 조기축구 매치 메이킹과 커뮤니티 매너 시스템을 포함한 종합 플랫폼으로 진화한다.

## 🚀 Phase 1: 플랫폼 기초 공사 (Auth & Persistence)
*가장 먼저 해결해야 할 기술적 토대입니다.*

### 1. SNS 가입 시스템 (Kakao, Google)
- **요구사항**: 사용자 편의를 위한 범용 소셜 로그인 구현.
- **작업 순서**: 
    1. OAuth2 프로토콜 기반 백엔드 인증 로직 설계.
    2. Kakao/Google Developer Console 앱 등록 및 API 키 설정.
    3. 프론트엔드 로그인 페이지 및 세션 관리 구현.
- **서브 작업**: 사용자 프로필 테이블(User Table) 설계 (이메일, 닉네임, 매너점수 포함).

### 2. 데이터베이스 전환 (JSON → RDBMS)
- **요구사항**: 다중 사용자 및 매치 관리를 위한 안정적인 데이터 구조 확보.
- **작업 순서**:
    1. PostgreSQL 또는 SQLite 스키마 설계.
    2. 기존 JSON 데이터 마이그레이션 스크립트 작성.
    3. SQLAlchemy/Prisma 등 ORM 연동.

## 🚀 Phase 2: 매치 메이킹 핵심 루프 (Match Management)
*서비스의 핵심인 '방 만들기'와 '참가' 기능을 구현합니다.*

### 3. 전체 매치 목록 및 검색 (Match Discovery)
- **요구사항**: 진행 전인 매치만 확인 가능하며 장소, 시간, 호스트 정보 제공. 비공개/공개 방 구분.
- **작업 순서**:
    1. 매치 목록 조회 API (필터링: 시간 > 현재 시각).
    2. 공개/비공개 방 필터링 UI 구현.
- **서브 작업**: 매치 상태(Status: 모집중, 완료, 취소) 관리 로직.

### 4. 참가 신청 및 호스트 승인 프로세스
- **요구사항**: 참가 신청 시 온보딩(포지션 입력 등) 실시, 호스트가 명단을 보고 수락 여부 결정.
- **작업 순서**:
    1. 참가 신청(Join Request) 테이블 및 API 구현.
    2. 호스트용 참가자 관리 대시보드 구현 (수락/거절 버튼).
    3. 신청자용 온보딩 폼 연동.

---

## 🚀 Phase 3: 운영 로직 및 UX 고도화 (Logic & UX)
*실제 사용 시 발생하는 불편함과 운영 정책을 반영합니다.*

### 5. AI 포지션 배정 로직 수정 (4-3-2-1 최적화)
- **요구사항**: 4-3-2-1 등 특정 포메이션에서 공격진 배치가 부자연스러운 문제 해결.
- **작업 순서**:
    1. 백엔드 `auto-assign` 알고리즘 내 공격 영역(FW) 좌표 및 우선순위 재검토.
    2. 포메이션별 슬롯 위치(XY 좌표) 상수값 정밀 조정.

### 6. 취소 정책 및 권한 제어 (Cancellation Policy)
- **요구사항**: 호스트는 경기일 특정 기간 전까지만 임의 취소 가능. 게스트는 마이페이지에서 취소.
- **작업 순서**:
    1. 매치 모델에 `cancellation_deadline` 필드 추가.
    2. 취소 가능 여부 체크 로직(현재 시간 vs 경기 시간) 구현.
    3. 권한별 UI 제어 (호스트/참가자 버튼 노출 여부).

### 7. 모바일 사용성 개선 (Guest-First View)
- **요구사항**: 게스트 접속 시 축구장 포지션보다 '참여자 명단'을 우선 노출.
- **작업 순서**:
    1. 반응형 레이아웃 재설계 (Mobile-specific priority).
    2. 역할(Host/Guest)에 따른 초기 탭 설정 변경.

---

## 🚀 Phase 4: 커뮤니티 및 사후 관리 (Social & Feedback)
*서비스의 유지보수성과 신뢰도를 높이는 단계입니다.*

### 8. 마이페이지 (Match History)
- **요구사항**: 지금까지 참여했던 모든 매치 이력 확인.
- **작업 순서**: 
    1. 사용자별 참여 완료 매치 쿼리 최적화.
    2. 매치 결과(승패 또는 기록) 표시 UI.

### 9. 매너 점수 시스템 (Manner Score)
- **요구사항**: 매치 종료 후 참가자 상호 평가 (당근마켓 방식). 유저 페이지에 노출.
- **작업 순서**:
    1. 평가 API 구현 (매치 종료 후에만 활성화).
    2. 유저 프로필 내 매너 점수(신뢰도) 계산 로직 및 시각화.
