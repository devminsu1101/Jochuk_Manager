# 조축매니저(Jochuk Manager) QA 분석 보고서 (2026.03.19)

본 문서는 현재 개발된 조축매니저 프로젝트의 코드 로직, 사용자 경험(UX), 그리고 배포 환경(Vercel) 기반의 잠재적 문제점을 심도 있게 분석한 결과입니다.

## 1. 핵심 로직 분석 및 버그 위험군

### 1.1 포메이션 변경 시 데이터 유실 (Frontend)
- **증상**: `useMatchStore.ts`의 `setFormation` 함수가 호출될 때 `assignedPlayers: {}`로 초기화됨.
- **원인**: 포메이션이 바뀌면 기존 슬롯 이름(예: `LCDM` -> `LCM`)이 달라지므로 단순히 초기화하도록 구현되어 있음.
- **영향**: 사용자가 공들여 짜놓은 라인업이 포메이션을 실수로 클릭하는 순간 모두 사라짐.
- **개선 제안**: 기존 선수들의 포지션을 유지할 수 있는 매핑 로직을 도입하거나, 변경 전 확인 팝업 필요.

### 1.2 AI 자동 배정의 공평성 한계 (Backend)
- **증상**: `auto_assign` 로직이 1회성 요청 내에서만 `playCount`를 계산함.
- **원인**: 백엔드 DB(`matches_db.json`)에 각 선수의 누적 `playCount`가 저장되지 않고 항상 0으로 유지됨.
- **영향**: 매번 자동 배정을 돌릴 때마다 이전 쿼터의 누적 데이터가 아닌, 현재 요청된 4개 쿼터 내에서의 비율만 맞추려 함. 또한, 수동으로 수정한 데이터는 백엔드가 알지 못하므로 자동 배정 시 중복 발생 가능성 있음.

### 1.3 하드코딩된 Match ID (`match-123`)
- **증상**: 프론트엔드와 백엔드 곳곳에 `match-123`이 고정값으로 들어가 있음.
- **원인**: 초기 개발 편의성을 위해 단일 매치로 고정.
- **영향**: 여러 팀이나 여러 날짜의 경기를 관리할 수 없음. URL의 `matchId` 파라미터가 실질적인 역할을 하지 못함.

## 2. 드래그 앤 드롭(DnD) 및 UX 이슈

### 2.1 쿼터 간 이동 불가
- **증상**: `handleDragEnd` 로직이 `if (fromQuarterId === targetQuarterId)` 조건으로 묶여 있어 다른 쿼터의 선수와 교체하거나 이동하는 것이 불가능함.
- **영향**: 1쿼터에 배치한 선수를 2쿼터로 옮기고 싶을 때 반드시 사이드바를 거쳐야 함.

### 2.2 사이드바 동기화 지연
- **증상**: `fetchPlayers`가 5초 간격으로 폴링(Polling) 수행.
- **영향**: 누군가 등록 페이지에서 선수를 추가했을 때 실시간으로 반영되지 않아 최대 5초간 대기해야 함. WebSocket 또는 더 짧은 간격의 최적화가 필요할 수 있음.

## 3. 아키텍처 및 배포 환경 이슈 (Vercel/Render)

### 3.1 파일 시스템 영속성 문제
- **증상**: 백엔드(`main.py`)가 `matches_db.json` 로컬 파일에 데이터를 저장함.
- **원인**: Vercel 등 서버리스 환경은 파일 시스템이 읽기 전용이거나 재배포/재시작 시 초기화됨.
- **영향**: 현재 Vercel에 배포된 상태라면, 서버가 재시작될 때마다 등록된 선수 목록이 초기화될 위험이 큼. (외부 DB인 MongoDB나 PostgreSQL로의 전환 권장)

### 3.2 CORS 및 환경 변수
- **증상**: `NEXT_PUBLIC_API_URL`이 설정되지 않으면 `localhost:8000`으로 기본 설정됨.
- **영향**: 실제 배포 환경에서 환경 변수 누락 시 통신 장애 발생.

## 4. UI/UX 세부 피드백

- **모바일 뷰 최적화**: 현재 `SoccerPitch`의 크기가 고정되어 있어 모바일 기기에서 좌우가 잘리거나 드래그가 어려울 수 있음.
- **공유 기능**: `html-to-image` 라이브러리는 모바일 브라우저(특히 iOS Safari)에서 이미지 생성 시 폰트 깨짐이나 렌더링 오류가 잦음. Web Share API 등을 고려해볼 필요가 있음.
- **자동 배정 알림**: 현재 `alert`로만 결과가 안내되는데, 어떤 기준으로 배정되었는지(예: 주 포지션 우선 등)에 대한 가이드가 있으면 신뢰도가 높아짐.
